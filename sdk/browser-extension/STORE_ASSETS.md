# Chrome Web Store — Required Assets

This file documents every asset the Chrome Web Store listing needs. Image files
cannot be generated by CI and must be created manually or by a designer.

---

## Screenshots

All screenshots must be **1280 x 800** or **640 x 400** PNG files. At least 1
is required; 5 are recommended for a polished listing.

| File name | Dimensions | Content |
|-----------|------------|---------|
| `screenshot-1-chat-input.png` | 1280 x 800 | ChatGPT or Claude page with input containing "My phone is 13812345678". Floating badge at bottom-right shows "3 entities detected". |
| `screenshot-2-masked-send.png` | 1280 x 800 | Same page after sending. AI response shows masked text with `[PII_PHONE_0001]` placeholder. |
| `screenshot-3-context-menu.png` | 1280 x 800 | Text selected on any page, right-click context menu open showing "Copy masked version" option. |
| `screenshot-4-options.png` | 1280 x 800 | Options page (`options.html`): gateway URL field, entity type checkboxes (13+), custom words textarea. |
| `screenshot-5-popup.png` | 640 x 400 | Popup (`popup.html`): "Connected" status, stats (entities masked today), master toggle. |

## Developer Dashboard Notes

When submitting via the Chrome Web Store Developer Dashboard:

1. **Single purpose:** The extension has one purpose — PII masking for AI chat
   platforms. State this clearly.
2. **Host permissions justification:** For each host pattern in `manifest.json`,
   explain why it's needed (e.g., `https://chat.openai.com/*` = ChatGPT support).
3. **Remote code:** The extension does not load any remote code — confirm this.
4. **Data usage:** All processing is local. No user data is collected or sent
   to external servers.
